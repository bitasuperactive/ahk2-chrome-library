◆ __New()

Chrome::Page::__New ( url ,
events : = 0 )
inline

Conecta con la interfaz de depuración de una página web determinada por su WebSocket URL.

  • Mantiene viva la conexión enviando una solicitud cada 25 segundos.
  • Se autodesecha al cerrar la pestaña.
    Parámetros
    {String}url URL WebSocket de la página a conectar.
    {Func}events (Opcional) Función a ejecutar cuando se reciba un mensaje de la página: msg => void.

◆ WaitForLoad()

Chrome::Page::WaitForLoad ( desiredState : = 'complete',
interval : = 100,
timeout : = 10000 )
inlineprivate

Espera a que el documento pase al estado indicado.

Nota
El documento pasa al estado "complete" sin finalizar la carga los elementos asíncronos. Si se requieren, utilizar WaitForElement.
Parámetros
{String}desiredState (Opcional) Estado deseado para el documento. Por defecto: "complete".
{Integer}interval Intervalo en milisegundos de espera entre las evaluaciones del estado.
{Integer}timeout (Opcional) Límite de tiempo (ms) para anular la búsqueda del elemento. Por defecto: 10000.
Devuelve
{Chrome.Page}
Excepciones
{Error}Si se alcanza el tiempo de espera sin que el documento alcance el estado deseado.
Ver también
https://www.w3schools.com/jsref/prop_doc_readystate.asp
